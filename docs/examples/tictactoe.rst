4x4x4 三目並べ (Tic-Tac-Toe)
==============================

このサンプルでは、BDDを用いて4x4x4立体三目並べの引き分け局面数を計算します。
ソースコード: ``examples/tictactoe.cpp``

問題の定義
----------

4x4x4の立体格子上で三目並べを行ったとき、指定個数のX（クロス）が配置された
引き分けの局面数を求めます。引き分けとは、XもOもどの勝利ラインも完成していない
状態を指します。

4x4x4立体格子には76本の勝利ラインがあります：

* 行方向: 16本
* 列方向: 16本
* 高さ方向: 16本
* 2D対角線: 24本
* 3D空間対角線: 4本

アルゴリズム
------------

BDDの各変数は格子の各マス（64マス）に対応し、変数が真ならX、偽ならOを表します。

1. **個数制約の構築**: ちょうどN個のXが配置されるBDDを動的計画法で構築
2. **勝利ライン制約の適用**: 各勝利ラインについて「すべてXでない」かつ
   「すべてOでない」という制約をANDで結合
3. **引き分け局面の計数**: 両制約を満たす割り当て数を ``count()`` で計算

使用API
-------

* ``DDManager``, ``BDD``
* ``mgr.var_bdd(v)`` -- 変数BDD
* ``BDD::count(n)`` -- 充足割当数
* 演算子: ``&`` (AND), ``|`` (OR), ``~`` (NOT)

実行方法
--------

.. code-block:: bash

   ./tictactoe        # デフォルト: N=20
   ./tictactoe -n 10  # X=10個の場合
