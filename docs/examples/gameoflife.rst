Game of Life
============

このサンプルでは、BDDを用いてConwayのGame of Lifeの遷移規則を表現し、
特定の性質を持つ初期配置の数を計算します。
ソースコード: ``examples/gameoflife.cpp``

問題の定義
----------

ConwayのGame of Lifeは、N x Nのグリッド上でセルの生死が以下のルールで
決定されるセルオートマトンです：

* **生存**: 生きているセルは、隣接する生きたセルが2個または3個なら次世代も生存
* **誕生**: 死んでいるセルは、隣接する生きたセルがちょうど3個なら次世代に誕生
* **過疎/過密**: 上記以外の場合、セルは死亡

このサンプルでは2つの問いに答えます：

1. 中央セルが次世代で生存する初期配置はいくつあるか
2. Still Life（全セルが不変な配置）はいくつあるか

アルゴリズム
------------

各セルに1つのBDD変数を割り当て（合計 N x N 個）、各セルについて
ライフゲームのルールをBDDで構築します。

1. **近傍カウント制約**: 各セルの隣接セル数を動的計画法でBDDに符号化
2. **ライフルール適用**: 各セルについて「現在の状態と近傍数に基づく次世代の状態」をBDDで表現
3. **中央セル分析**: 中央セルのライフルールBDDの充足割当数を計算
4. **Still Life**: すべてのセルのライフルールをANDで結合し、充足割当数を計算

使用API
-------

* ``DDManager``, ``BDD``
* ``mgr.var_bdd(v)`` -- 変数BDD
* ``BDD::count(n)`` -- 充足割当数
* 演算子: ``&`` (AND), ``|`` (OR), ``~`` (NOT)

実行方法
--------

.. code-block:: bash

   ./gameoflife       # デフォルト: 3x3
   ./gameoflife -n 4  # 4x4グリッド
